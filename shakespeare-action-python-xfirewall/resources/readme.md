# XFirewall 新一代网络防火墙

> **版本**: v1.0.2
> **发布日期**: 2026年1月
> **兼容平台**: Octomation SOAR v1.0+

## 应用简介

XFirewall 是一款现代化的网络防火墙管理系统，专为 Octomation SOAR 平台设计。它提供了简洁高效的 IP 地址封禁和管理功能，支持实时威胁响应和自动化安全编排。

## 主要功能

### 🛡️ IP 封禁管理
- **IP 封禁**: 快速封禁恶意 IP 地址，支持自定义封禁原因
- **IP 解封**: 灵活解除对 IP 地址的封禁限制
- **封禁记录查询**: 实时查看当前活跃的封禁规则
- **历史记录追溯**: 完整的操作历史记录和审计轨迹
- **策略提交模拟**: 封禁和解封时模拟 1-3 秒设备策略下发延迟，并返回提交 ID、下发节点和同步状态

### 🔍 智能查询
- 支持按 IP 地址、时间范围进行精确查询
- 分页显示，处理大量数据
- 实时状态监控和健康检查
- 查询管理平面、策略引擎、威胁情报库、日志管道和本地规则库状态

### 🗄️ 数据持久化
- 内置 SQLite 数据库，确保数据安全可靠
- 完整的操作日志记录和审计轨迹
- 支持分布式部署环境（配合 NFS 共享存储）

## 配置参数

### 必需配置
- **防火墙地址** (`firewall_url`): 防火墙管理接口地址
- **管理令牌** (`admin_token`): 具有管理权限的 API Token

### 可选配置
- **数据库路径** (`database_path`): SQLite 数据库文件存储位置
  - 默认: `/tmp/xfirewall.db`
  - 分布式环境建议使用 NFS 共享路径
- **最小模拟延迟** (`delay_min_seconds`): 封禁/解封操作的最小模拟延迟秒数，默认 `1`
- **最大模拟延迟** (`delay_max_seconds`): 封禁/解封操作的最大模拟延迟秒数，默认 `3`

## API 接口

### 1. 封禁 IP (block_ip)
封禁指定的 IP 地址，支持自定义封禁原因。

**参数：**
- `ip`: 需要封禁的 IP 地址（必填）
- `reason`: 封禁原因（可选，默认：安全策略封禁）

**返回：**
- `result`: 操作结果描述
- `rule_id`: 封禁规则的唯一标识
- `policy_commit`: 策略提交信息，包含提交 ID、同步状态、下发节点和模拟耗时
- `debug_info`: 调试信息（包含数据库路径和时间戳）

### 2. 解封 IP (unblock_ip)
解除对指定 IP 地址的封禁。

**参数：**
- `ip`: 需要解封的 IP 地址

**返回：**
- `result`: 操作结果描述（包含调试信息）
- `unblocked_count`: 解封的规则数量
- `policy_commit`: 策略提交信息，包含提交 ID、同步状态、下发节点和模拟耗时

### 3. 查询封禁列表 (query_blocked_ips)
获取当前活跃的封禁 IP 列表。

**参数：**
- `limit`: 返回结果数量限制（可选，默认100）

**返回：**
- `blocked_ips`: 封禁 IP 详细信息数组
- `total_count`: 总封禁 IP 数量
- `debug_info`: 调试信息（数据库路径、查询限制等）

### 4. 查询历史记录 (query_block_history)
查询封禁和解封的历史操作记录。

**参数：**
- `ip`: 要查询的 IP 地址（可选）
- `start_time`: 开始时间（可选）
- `end_time`: 结束时间（可选）
- `limit`: 返回结果数量限制（默认100）

**返回：**
- `history_records`: 历史操作记录数组
- `total_count`: 总记录数量

### 5. 查询防火墙状态 (query_firewall_status)
查询防火墙总体运行状态和关键组件状态。

**返回：**
- `status`: 防火墙运行状态
- `health_level`: 健康等级
- `components`: 关键组件状态列表
- `system_info`: 系统指标、策略版本、威胁库版本和资源使用情况

### 6. 健康检查 (health_check)
检查防火墙系统的运行状态和健康情况。

**返回：**
- `msg`: 健康检查结果信息
- `system_info`: 系统状态详细信息
  - 版本信息
  - 运行状态
  - 系统资源使用情况
  - 数据库连接状态
  - 关键组件状态

## 使用场景

### 威胁情报集成
当威胁情报系统发现恶意 IP 时，可自动调用封禁接口进行实时防护：
```
1. 威胁情报系统 → XFirewall封禁接口
2. 自动记录封禁原因和时间
3. 生成操作日志用于审计
```

### 安全事件响应
结合 SOAR 平台的自动化流程，实现快速威胁响应：
```
1. 安全事件触发 → 自动化剧本
2. IP 信誉检查 → 风险评估
3. 自动封禁 → 威胁隔离
4. 通知安全团队 → 人工审核
```

### 定期安全维护
支持批量操作和定时任务：
```
1. 定期查询封禁列表
2. 清理过期封禁规则
3. 生成安全报告
4. 系统健康检查
```

## 数据库结构

### blocked_ips 表
存储当前封禁规则信息：
- `id`: 规则唯一标识
- `ip_address`: 被封禁的 IP 地址
- `reason`: 封禁原因
- `created_time`: 创建时间
- `status`: 规则状态（active/removed）
- `rule_hash`: 规则哈希值（防重复封禁）

### operation_logs 表
记录所有操作历史：
- `operation_type`: 操作类型（block/unblock）
- `ip_address`: 操作的 IP 地址
- `reason`: 操作原因
- `operator`: 操作者
- `timestamp`: 操作时间
- `result`: 操作结果

## 安全特性

- **输入验证**: 严格验证 IP 地址格式，防止恶意输入
- **操作审计**: 完整记录所有操作日志，支持安全审计
- **数据隔离**: 独立的数据库实例，确保数据安全
- **权限控制**: 基于 Token 的访问控制机制

## 部署建议

### 单机部署
1. **生产环境**: 建议将数据库文件存储在持久化存储卷中
2. **备份策略**: 定期备份 SQLite 数据库文件
3. **监控告警**: 集成系统监控，及时发现异常情况
4. **日志管理**: 配置日志轮转，避免日志文件过大

### 分布式部署
1. **共享存储**: 使用 NFS 或其他网络共享文件系统存储数据库
2. **数据库路径**: 配置为共享路径，如 `/nfs/shared/xfirewall.db`
3. **一致性保证**: 确保所有节点访问同一个数据库文件
4. **网络性能**: 优化网络连接，降低数据库访问延迟
